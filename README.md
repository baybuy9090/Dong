# 동업계 남성 컨템 브랜드 입점 현황

롯데·현대·신세계 남성 컨템포러리 브랜드 37종의 45개 지점 입점 현황을 매달 자동으로 크롤링하고, 웹 대시보드로 보여주는 도구입니다.

## 구조

- `crawler.js` — 크롤링 스크립트 (Node.js)
- `.github/workflows/crawl.yml` — 매달 1일 자동 실행 (수동 실행도 가능)
- `data.json` — 크롤링 결과 (자동 생성/갱신됨)
- `truth.json` — 정답 대조용 데이터 (직접 채워야 함, 아래 참고)
- `index.html` — 대시보드 화면

## 처음 설정하는 방법

1. **저장소 만들기**
   - GitHub에서 새 저장소 생성 (Public)
   - 이 5개 파일을 그대로 올리기 (`crawler.js`, `.github/workflows/crawl.yml`, `data.json`, `truth.json`, `index.html`)

2. **GitHub Pages 켜기**
   - 저장소 → Settings → Pages
   - Source: "Deploy from a branch" → Branch: `main`, 폴더: `/ (root)` → Save
   - 잠시 후 `https://[사용자명].github.io/[저장소명]/` 에서 대시보드 확인 가능

3. **크롤링 최초 실행**
   - 저장소 → Actions 탭 → "월간 브랜드 크롤링" 워크플로우 선택
   - "Run workflow" 버튼으로 수동 실행 (완료까지 몇 분 소요, GAS와 달리 6분 제한 없음)
   - 완료되면 `data.json`이 자동으로 커밋됨 → 대시보드 새로고침하면 반영

4. **정답 대조 데이터 채우기 (선택)**
   - `truth.json`을 아래 형식의 배열로 채우기:
   ```json
   [
     { "company": "롯데", "store": "본점", "brand": "타임옴므" },
     { "company": "롯데", "store": "본점", "brand": "스톤아일랜드" }
   ]
   ```
   - "맨즈 컨템 현황" 시트에서 1로 표시된 (회사, 지점, 브랜드) 조합을 전부 이 형식으로 옮기면 됩니다.
   - 비워두면(`[]`) 정답 대조 없이 크롤링 결과만 보여줍니다.

## 이후 사용

- 매달 1일 09:00(KST)에 자동으로 재크롤링되고, `data.json`이 자동 갱신됩니다.
- 급하게 다시 돌리고 싶으면 Actions 탭에서 "Run workflow"로 수동 실행 가능합니다.
- `index.html`을 수정하고 싶으면 그냥 파일을 고쳐서 커밋하면 됩니다 (배포 버전 갱신 절차가 따로 없습니다).

## 알려진 한계

- 롯데 크롤링의 `townCd`는 강남점 기준값이라, 다른 지점에서는 정확하지 않을 수 있습니다.
- "매출" 데이터는 각 사 홈페이지에 공개되어 있지 않아 자동 수집이 불가능합니다.
- 페이지 전체 텍스트에서 브랜드명을 찾는 방식이라, 실제 매장이 아닌 다른 영역(추천 브랜드 등)의 언급도 잡힐 수 있습니다.
